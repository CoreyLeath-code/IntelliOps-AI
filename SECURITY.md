# Security Policy

## Supported Versions

Security fixes are applied to the latest release and the `main` branch.

## Reporting a Vulnerability

Do not disclose suspected vulnerabilities in a public issue. Use GitHub private vulnerability reporting when available, or contact the repository owner privately with the affected component, reproduction steps, impact assessment, and any suggested mitigation.

Reports will be acknowledged as soon as practical. Confirmed issues are prioritized by severity and documented after remediation is available.

## Automated Controls

IntelliOps-AI uses CodeQL, Gitleaks, Trivy filesystem scanning, dependency auditing, Dependabot, CycloneDX SBOM generation, multi-language CI, container build validation, and release-readiness checks.
